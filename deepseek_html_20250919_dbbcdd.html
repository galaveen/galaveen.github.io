GitHub Pages Build Error

Fixing the "No such file or directory @ dir_chdir" error in GitHub Pages

Error Detected: Your GitHub Pages build is failing with the error:
Error: No such file or directory @ dir_chdir0 - /github/workspace/docs

This usually happens when GitHub Pages can't find your site's source files.

View Solutions

How to Fix This Error

Try these solutions to resolve the GitHub Pages build error

Check Your Source Folder

GitHub Pages is looking for a /docs folder but can't find it. Make sure your files are in the correct location.

Configure Source Settings

Go to your repository Settings → Pages → and select the correct source (root or /docs folder).

Add Index File

Ensure you have an index.html file in the root of your selected source directory.

Step-by-Step Troubleshooting

Follow these detailed steps to resolve the issue

Solution 1: Check your repository structure

The error suggests GitHub Pages is looking for a /docs folder but can't find it. Here's how to fix this:

  • If your files are in the root of your repository, change the GitHub Pages source to "Deploy from root"
  • If you want to use a /docs folder, create one and move your site files there
  • Make sure your index.html is in the correct location

To change the source: Go to your repository → Settings → Pages → Source → Select the appropriate option.

Solution 2: Check for configuration files

If you're using Jekyll (GitHub Pages' default site builder), you might have a configuration issue:

  • Check for a _config.yml file in your repository
  • Ensure it doesn't contain incorrect settings that might break the build
  • If you're not using Jekyll, add an empty .nojekyll file to your root directory

Sometimes, disabling Jekyll can resolve build issues for simple HTML sites.

Solution 3: Check your branch and file structure

Your GitHub Pages deployment might be looking at the wrong branch or folder:

  • Ensure you're using the correct branch (usually main or gh-pages)
  • Verify that your files are actually in the repository and committed
  • Check that filenames are using consistent casing (Git is case-sensitive)

To change the branch: Go to your repository → Settings → Pages → Branch → Select the correct branch.

Setting Up GitHub Pages

Follow these steps to correctly configure your GitHub Pages site

1

Repository Name

Name your repository username.github.io (replace username with your GitHub username)

2

Add Index File

Ensure you have an index.html file in the root of your repository

3

Enable Pages

Go to Settings → Pages → Select main branch as source

4

Wait for Deployment

It can take up to 10 minutes for your site to be published

Troubleshooting Guide

Common issues and solutions for GitHub Pages "File Not Found" errors

Incorrect repository name

Your repository must be named exactly username.github.io where "username" is your GitHub username. This is case-sensitive.

To fix: Rename your repository or create a new one with the correct naming convention.

Index file missing or misnamed

You must have an index.html file in the root of your repository. The filename is case-sensitive.

To fix: Ensure your main HTML file is named exactly index.html and located in the root directory.

GitHub Pages not enabled

You need to enable GitHub Pages for your repository.

To fix: Go to your repository → Settings → Pages → Select "main" or "gh-pages" as source branch.

Deployment in progress

It can take a few minutes for your site to deploy after pushing changes.

To fix: Wait 5-10 minutes and check again. You can see the deployment status in the "Actions" tab.

Custom domain misconfiguration

If using a custom domain, you need to configure it properly in your repository settings and DNS.

To fix: Check GitHub's documentation for custom domain setup and verify your DNS records.

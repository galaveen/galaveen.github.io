Jekyll SCSS Conversion Error

Fixing the "No such file or directory @ dir_chdir" error for SCSS files in Jekyll

Error Detected: Your Jekyll build is failing with the error:
Conversion error: Jekyll::Converters::Scss encountered an error while converting 'assets/css/style.scss': No such file or directory @ dir_chdir0 - /github/workspace/docs

This usually happens when Jekyll can't find the referenced SCSS file or its dependencies.

View Solutions

How to Fix This Error

Try these solutions to resolve the Jekyll SCSS conversion error

Check File Paths

Jekyll can't find your SCSS file. Make sure the file path is correct and the file exists in your repository.

Verify Directory Structure

Ensure your SCSS files are in the correct directory. Jekyll expects certain folder structures for assets.

Check Front Matter

SCSS files need proper front matter to be processed by Jekyll. Add YAML front matter if missing.

Step-by-Step Troubleshooting

Follow these detailed steps to resolve the SCSS conversion issue

Solution 1: Verify the SCSS file exists

The error suggests Jekyll can't find your SCSS file. Here's how to fix this:

  • Check that the file assets/css/style.scss exists in your repository
  • Verify the file path is correct (case-sensitive)
  • Ensure the file has been committed to GitHub
  • If the file is in a different location, update references to it

To check: Go to your repository on GitHub and navigate to the assets/css directory to see if style.scss exists.

Solution 2: Add front matter to SCSS files

Jekyll requires SCSS files to have front matter to process them:

  • Open your SCSS file (e.g., assets/css/style.scss)
  • Add this to the very top of the file:
---
---
  • Save the file and commit to GitHub
  • This tells Jekyll to process the file

Without this front matter, Jekyll will ignore the SCSS file, which can cause build errors.

Solution 3: Check for import statements

The error might be caused by missing imported SCSS files:

  • Check your style.scss file for @import statements
  • Ensure all imported files exist in the correct locations
  • Verify import paths are correct
  • Make sure imported files also have front matter if they need processing

Example of correct import:

@import "variables";
@import "mixins";

Make sure variables.scss and mixins.scss exist in the same directory.

Solution 4: Check your _config.yml

Your Jekyll configuration might be causing the issue:

  • Check your _config.yml file for Sass/SCSS settings
  • Look for sass_dir configuration
  • Ensure the configured directory exists
  • Try simplifying your configuration to isolate the issue

Example configuration:

sass:
  sass_dir: assets/css
  style: compressed

Make sure the sass_dir value matches your actual directory structure.
